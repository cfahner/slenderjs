Popups and dialogs

Popups are UI elements that display another page within the current page. Define a popup by adding the class sl-popopen to an A tag with an href attribute. The target page should contain an element called sl-popclose to close the popup from the popup itself. When Javascript is disabled, or when the screen is too small, the link is followed.

Specify on which screens a popup should be opened by adding the data-screen-width attribute. This attribute takes either a literal dimension (in ems or pixels) or the keywords medium and large.

An alternative popup-only URL to follow can be specified using the data-pophref attribute.

Sending data back from popups

Data can be sent from another popup using the .sl-popyield class. Elements with this class send data back to the element that opened the initial popup by triggering the slpopyield event on it. The data sent is determined by the values of the @data-popyield and @data-popyield-type attributes (type is optional).

In a popup that allows "unselecting" items, the popup could yield data of a different type to indicate a deselection (changing the value of @data-popyield-type after selection).

Submitting forms to popups

Add the class sl-popopen to a form to automatically submit that form to a new popup (the default submit action is ignored). The popups API will automatically use the action and method attributes of the form to determine the request type and URL.

The enctype attribute is not supported.
